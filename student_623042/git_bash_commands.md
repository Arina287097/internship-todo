1. git clone <url> - клонирует репозиторий с удаленного сервера на локальную машину.
2. mkdir <name> - создает новый каталог.
3. echo "<text>" > README.md - создает файл README.md и записывает в него текст.
4. touch .gitignore - создает пустой файл .gitignore.
5. git add . - добавляет все изменения в текущем каталоге к следующему коммиту.
6. git commit -m "<message>" - фиксирует изменения с описанием.
7. git push origin main - отправляет коммиты на удаленный репозиторий.

Файл .gitignore необходим для указания Git, какие файлы или каталоги следует игнорировать при коммитах. Это полезно для исключения временных файлов, логов, конфиденциальных данных и других ненужных файлов.
